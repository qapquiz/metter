  appear in production.

## Findings considered and rejected

- **Wrapping all 19 endpoints now** — rejected: the user explicitly scoped early
  runs to `portfolio/open`. Future endpoints become one method + one types
  block on top of the existing structure.
- **Adding a runtime HTTP/throttle/retry/caching dependency** — rejected: the
  wrapper uses the global `fetch` (Node 18+/Bun/Deno/browsers) with no HTTP
  client lib, and does no throttling/retrying/caching/auth. `zod` is the SOLE
  runtime dependency, added on explicit user request for runtime response
  validation. No other deps. (Direction option D3 reconsiders retry as an
  opt-in feature; it is not a rejected finding, just not chosen yet.)
- **Automated pagination iterator / async generator** — rejected as a default:
  the response already exposes `page`/`pageSize`/`hasNext`/`totalCount`, so
  callers can paginate manually. Deferred to D2 if/when desired.
- **The pre-existing CI/release findings from the prior audit pass** (npm
  publish missing `NODE_AUTH_TOKEN`, unpinned `bun-version: latest`, the
  optional `typescript` peer dep) — the npm-auth one is plan 003 (DONE,
  merged `75b10e4`); the other two are plans 004 and 005.
- **2026-06-21: combining F2 + F5 into a single plan** — rejected: they touch
  disjoint files (`workflows/*.yml` vs `package.json`) and are independently
  reviewable; kept as separate plans 004 and 005.
